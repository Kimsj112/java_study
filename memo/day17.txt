스트림(Stream)
- 배열, 컬렉션 같은 데이터 묶음을 편하고 효율적으로 처리하기 위해 제공되는 API
- 선언형(함수형) 프로그래밍을 자바로 구현해 놓은 클래스
- 람다를 활용할 수 있는 기술
- 스트림은 값의 흐름을 의미함.

선언형 프로그래밍이란?
  - 명령형 프로그래밍 : 어떻게 할 것인가? (알고리즘에 중점)
   for(int i=0; i<ar.length; i++) {
      System.out.println(ar[i] + 1);
   }

  - 선언형 프로그래밍 : 무엇을 할 것인가? (목표에 중점)
   Arrays.stream(ar).forEach(num -> System.out.println(num+1));

자바의 선언형 프로그래밍 지원
	- JDK8부터  Strean과 lambda를 지원한다.
	- stream에서만 사용 가능한 메소드를 이용하여 프로그래밍 한다.
	- 마치 블록을 조립하듯 메소드를 나열하여 프로그래밍 한다.(메서드 체이닝)

스트림의 종류
	- 기본타입 Stream
		IntStream : int타입의 데이터를 처리하기 위한 스트림
		DoubleStream : ""
		LongStream : ""

	- 객체타입 Stream
		Stream<T> : T타입의 데이터를 처리하는 스트림

스트림의 3단계
	1. 생성
		- 범위를 이용한 생성
		- Arrays를 이용한 생성
		- Collection을 이용한 생선
	2. 중간 연산
		- 반환타입이 stream임.
		- 몇 번이든 사용 가능.

	3. 최종 연산
		- 반환타입이 stream이 아님.
		- 마지막에 한번만 사용 가능함.

스트림의 특징
	1. 스트림은 일회용임. -> 재사용 불가능
	2. 스트림은 원본 데이터를 수정하지 않음.
